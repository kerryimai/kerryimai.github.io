Hey there! This week I am going to discuss the differences between Ruby Classes and Javascript constructors functions. If you have read some of my other blogs, this would probably remind you the one from last week : Classy Ruby Classes! If you have not, no worries, I will go through a review with you shortly.
As you might be the many among us who thought Javascript constructors were Classes, this might come as a bit of a shock, but jsut know that these two things are actually quite similar and the differences are easily to extinguishable. A Ruby class itself can be seen as an object. However it is a special kind of object that contains special methods and characteristics. It as two main functions:

    ---creating new instances(that falls into this specific class)

    ---defines the behavior of the instances

    Here is an example of these two functions in action:

        

      class Artists

        def initialize(first_name, last_name, artist_type)
          @first_name = first_name
          @last_name = last_name
          @artist_type = artist_type
        end

        def introduce
          "#{@first_name} #{@last_name} is a
          famous #{@artist_type}"
        end
      end

      art1 = Artists.new("Beyonce", "Knowles", "singer")
      puts art1.introduce

Above, with the initialize code, I am able to create an new instance 'art1' which belongs in the Artists class. Thus when I call the instance method "introduce" on my art1, it automatically plugs in the arguemetns from the initailize call and returns the result.

Now you are probably thinking: Doesn't Javascript constructors do the same thing? It does appear so but there ar actually sufficient differences between the two. Look at this image below, your initial thought might be thinking that it was an owl, but when you look closer into the details, its actually made with very different things.

Just like this cup of owl looknig coffee, in Javacript, the illusion of what seemingly to be the function of class is split in two things: Constructor and Prototype.
---Constructors creat new instances
---Prototypes are used to define the behaviour of instances.
JavaScript objects don’t have a formal class, and thus there’s no special syntax for defining how to create an instance or define its behaviour: 

        


        function Artists (firstName, lastName, artistType) {
          this.firstName = firstName;
          this.lastName = lastName;
          this.artistType = artistType;
        };

        Artists.prototype.introduce = function () {
          return this.firstName + " " + this.lastName +
          " is a famous " + this.artistType;
        };

        var art1 = new Artists("Beyonce", "Knowles", "singer");
        console.log(art1.introduce());

As you would notice from the example above that the constructor is a function, any function can be a potential constructor in Javascript. That is what is different from Class, as Class has special syntax and it is a special kind of object. Nevertheless, unlike Ruby Classes, there are no special methods or properties associated with a prototype. Any object can be a prototype, even an empty object! I hope this helps you understand a little more about the difference and function of Ruby Classes and Javascript Constructor Functions See you next week.
