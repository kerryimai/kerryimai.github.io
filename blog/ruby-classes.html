This week's post is about the powerful use of classes in Ruby. Ruby classes would become very important when you start to write codes that are larger and more complex.The bigger the data, the harder it is for you to keep track of everything manually. Using classes would help you become able to recycle your methods and also stores all your relative data under one place.Look at it as your closet in your apartment. It helps you organize all your clothings into one place. In this post, I will use Closet as your class, and make a program to help you organize your clothing items inside your closet.

So far, we have been pretty familiar of writing a variety of useful methods. But they have all been independent from each other. What if you would like to use several of these methods on one item without having to repetiively inputing the same data? This is where class would come to be useful.

Above, I created a method inside the class where you can keep track of how many times you have used an items in your closet. Whenever you decide to take something out and wear it, just type the command: object.used_it like I did above, and ruby will help you keep track of the used record. So far, we have set up a pretty good system, but for now you wont be able to access the data of the items from outside the method. To do that, you would need to use a method called > attr_accessor, and pass the symbal of the names you would want to access:

      class Closet
        attr_accessor :color, :type, :date_bought, :times_used
        def initialize(color, type, date_bought, times_used)
          @color = color
          @type = type
          @date_bought = date_bought
          @times_used = times_used
        end
        def used_it
          @times_used += 1
        end
      end
    #So now when I call and ask data from the outside of the class:
