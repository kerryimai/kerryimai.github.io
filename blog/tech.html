During this 8 weeks of learning how to code, I have heard the word "Recursion" again and again. However, maybe its becasue of the complexity of this topic, I havn't yet offcially encountered it in programming. Thus this week, out of curiosity, I've decided to do some research and understand the idea of recursion in the programming language.

Recursive programming is the process of progressively reducing a problem in to easier to solve versions of itself. It is a lot like a mathmetical function. Except it calls a function from within that same function.

      

        Every recursive function tends to:
        - take a list to process, or some other
        structure, or problem domain
        - deal with the current point/step
        - call itself on the remainder(s)/subdomain(s)
        - combine or use the results of the subdomain work

Let's take the real world 'operation' of cutting up a pizza. You've taken the pizza out of the oven and in order to serve it up you have to cut it in half, then cut those halves in half, then again cut those resultant halves in half. The operation of cutting the pizza you performing again and again until you've got the result you want (the number of slices). Let's say that an uncut pizza is a slice itself. 

      

        def cut_pizza(existing_slices, desired_slices)
          if existing_slices != desired_slices
            # we don't have enough slices yet to feed everyone, so
            # we're cutting the pizza slices,
              thus doubling their number
            new_slices = existing_slices * 2
            # and this here is the recursive call
            cut_pizza(new_slices, desired_slices)
          else
            # we have the desired number of slices, so we return
            # here instead of continuing to recurse
            return existing_slices
          end
        end

        pizza = 1 # a whole pizza, 'one slice'
        cut_pizza(pizza, 8) # => we'll get 8

So the real world operation is cutting a pizza, and the recursion is doing the same thing over and over until you have what you want. Other tasks you'll find that can use recursive functions include:
- Calculating compound interest over a number of months.
- Looking for a file on a file system
It's simply programming mechanics, a way of cleverly removing duplication.
