    4.3 Variables and Methods (medium)

    Summary

    Now is your chance to put what you've learned together!

    Release 0: Mixing it up

    Read Chris Pine, Learn to Program, 2nd ed. Chapter 5, Mixing it Up

    I can't tell you how many times you will (probably) make the mistake of trying to perform operations on Objects that don't have that method - or just being confused about what kind of object something is all the time. It's incredibly important to know which kind of variable you are working with. If you don't know, you will get some crazy errors. Use IRB to practice as you read through it.

    Release 1: Prepare 

    Using your command line:

    • Navigate to your phase-0 repository (make sure it's up to date)
    • Create a new feature branch
    • Navigate to the week-4 directory
    • Create a file called "variables-methods.rb"

    Release 2: Try it out

    Complete the two mini-challenges Pine created (outlined below) in your variables-methods.rb file in Sublime and run it through your terminal. 

    • Full name greeting: Write a program that asks for a person's first name, then middle, then last. Finally, it should greet the person using their full name.

    • Bigger, better favorite number: Write a program that asks for a person's favorite number. Have your program add 1 to the number, and then suggest the result as a bigger and better number.

    Release 2: Creating Methods

    It's time to deviate from Pine's introduction by introducing you to the idea of creating your own methods.

    You can assign blocks of code to an object called a method. This encapsulates code and will run when you call the name of the method instead of each line of code each time. For example, imagine writing return addresses on envelopes (who does that anymore)? Instead of writing it out every. single. time., it would be easier to create a label. This is basically the same idea as using methods. If you were to use Pine's instructions to get an address from a user, it could look something like this:

    puts "What is your name?"
name = gets.chomp
puts "What is your street address?"
street_address = gets.chomp
puts "What is your city, state, and zip code?"
city_state_zip = gets.chomp
puts #this will add a new line
puts name
puts street_address
puts city_state_zip

    That's a lot of code for one simple address!

    Here is how you can use a method to "wrap" the code. You are changing how the code works a bit here, instead of getting user input for the arguments, you are defining them when you call the method.

    # print_address is a method we've defined
# print_address takes four arguments (name, street_address, city_state, zip)

def print_address(name, street_address, city_state, zip)
  puts name
  puts street_address
  puts city_state + zip.to_s  #the zip has been converted to a string
end

print_address("Maria Piper", "633 Folsom Street", "San Francisco, CA", 94107)
# this last statement calls the method which will output to the screen

    It's worth noting, Pine uses different syntax for methods, but it works just the same. Take the print_address method above. It can also be written like this:

    # Note the space and lack of parenthesis
def print_address name, street_address, city_state, zip
  puts name
  puts street_address
  puts city_state + zip.to_s
end

    Some programmers prefer the former, and others prefer the latter. Both are fine. It's similar to how you can use single quoted or double quoted strings. It's about preferences.

    In the next few exercises, we will be asking you to "wrap" logic into a method.

    Now's your chance to practice!

    In your command line, 

    • Create a new directory called define-method
    • Create the following files:
      • my_solution.rb
      • defining_variable_spec.rb
    • Copy the code from each file (Links to an external site.) to the corresponding file you created.

    In the my_solution.rb file, define a method called do_something that accepts three parameters. You do not need to write any code in the body of the method. Check that your code is valid by running the tests as follows:

    rspec spec-file-name.rb #using the filename of the spec file.

    NOTE: In the earlier challenges, the RSpec was included in the file itself. We did that out of necessity; it's not great practice. Use the format above to run tests for all future exercises. If you run the tests properly, there will always be a number of tests listed. If it says "0 examples, 0 failures", you probably ran the my_solution file rather than the spec file, so re-run it with the spec file.

    Release 3: More About Methods

    Read Chris Pine, Learn to Program, 2nd ed. Chapter 6: More About Methods.

    Read through the entire chapter to get a better feel for the built-in methods Ruby has. (There are a LOT more in the Ruby Docs (Links to an external site.), by the way - feel free to glance at them but no need to get too overwhelmed by everything that is there yet) .

    Release 4: More small exercises!

    Expand your brain a bit by working on the following exercises:

    • Return a Formatted Address
    • Defining Math Methods

    Release 5: Create a Pull Request

    Go through your Git Workflow to create a pull request of all of the files you created for this challenge. 

    Release 6: Reflect

    In the variables-methods.rb file you created, add a comment at the bottom and answer the following questions:

    • How do you define a local variable?
    • How do you define a method?
    • What is the difference between a local variable and a method?
    • How do you run a ruby program from the command line?
    • How do you run an RSpec file from the command line?
    • What was confusing about this material? What made sense?

    Add a link to each exercise solution file as well so your peers can easily look at your code.

    Release 7: Submit

    Submit the link to your variables-methods.rb file on GitHub. The URL should look like this:

    https://github.com/[USERNAME]/phase-0/blob/master/week-4/variables-methods.rb (Links to an external site.)

     

    Release 8: Read (mandatory)

    Read chapter 2 of Well-Grounded Rubyist on "Objects, methods, and local variables." This is not about memorizing, rather, we want you to deepen your understanding of what you've already learned.
